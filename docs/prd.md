# PatPat 心理按摩对话 Bot — PRD

## 1. 需求背景和目标

用户在日常生活中会遇到情绪低落、焦虑、压力大等情况，但不一定有条件或意愿去找专业心理咨询师。PatPat 旨在提供一个低门槛、随时可用的情绪疏导对话工具，让用户用自然语言倾诉，获得共情式回应和心理疏导建议。

核心目标：
- 用户能用简单的话描述自己的问题，Bot 自动识别情绪并给出温暖、有针对性的回应
- 支持多轮持续对话，而非一问一答
- 安全底线：识别严重心理危机时引导用户寻求专业帮助

## 2. 用户画像和使用场景

用户画像：
- 18-40 岁，有基本手机/电脑操作能力
- 遇到情绪问题时倾向于先自己消化，不愿立即找人倾诉
- 对心理咨询有一定认知但门槛感较高

典型使用场景：
- 深夜加班后情绪低落，打开 PatPat 倾诉工作压力
- 与朋友/伴侣发生矛盾，想找人说说但不想麻烦别人
- 考试/面试前焦虑，需要情绪疏导
- 日常心情记录和自我觉察

## 3. 功能需求列表（按优先级排序）

P0（MVP 必须）：
1. 多轮对话能力 — 基于 LLM，维护上下文，支持持续交互
2. 情绪识别与共情回应 — 通过 system prompt 工程让 Bot 具备心理疏导风格
3. 终端 CLI 交互界面 — 最小可用产品，先跑通核心对话链路
4. 对话历史保持 — 单次会话内记住上下文

P1（重要）：
5. 危机识别与转介 — 检测到自伤/自杀等关键词时，输出专业求助热线信息
6. 对话历史持久化 — 将对话存储到本地文件，下次可继续
7. 情绪标签记录 — 每轮对话自动标记用户情绪（如：焦虑、悲伤、愤怒）

P2（锦上添花）：
8. Web UI 界面 — 提供浏览器访问的聊天界面
9. 情绪趋势可视化 — 基于历史对话的情绪变化图表
10. 多种疏导策略 — 支持认知行为疗法(CBT)、正念引导等不同风格切换

## 4. 非功能需求

- 性能：单轮回复延迟 < 3 秒（取决于 LLM API 响应速度），支持流式输出
- 安全：
  - 不存储用户敏感信息到远程服务器，对话数据仅保存在本地
  - API Key 通过环境变量管理，不硬编码
  - 危机关键词检测必须在 MVP 阶段就考虑
- 可用性：
  - 启动命令简单（一条命令即可运行）
  - 对话体验流畅，支持 Ctrl+C 优雅退出
- 可维护性：
  - 模块化设计，LLM 提供商可替换
  - Prompt 模板与代码分离

## 5. 技术方案建议

技术栈：
- 语言：Python 3.10+
- LLM 接入：OpenAI API（gpt-4o / gpt-3.5-turbo），通过 `openai` SDK
- CLI 交互：`rich` 库美化终端输出
- 数据存储：本地 JSON 文件存储对话历史
- 配置管理：`python-dotenv` 读取 `.env` 文件中的 API Key

项目结构：
```
patpat/
├── .env.example          # API Key 配置模板
├── requirements.txt      # 依赖
├── main.py               # 入口，CLI 交互循环
├── patpat/
│   ├── __init__.py
│   ├── bot.py            # 核心对话逻辑，管理消息上下文
│   ├── prompts.py        # System prompt 模板
│   ├── crisis.py         # 危机关键词检测
│   └── storage.py        # 对话历史持久化
└── README.md
```

核心实现思路：
1. `prompts.py` — 定义心理疏导风格的 system prompt，指导 LLM 以共情、温暖的方式回应
2. `bot.py` — 封装 OpenAI ChatCompletion 调用，维护 messages 列表实现多轮对话
3. `crisis.py` — 关键词匹配 + LLM 辅助判断，检测到危机时插入求助信息
4. `main.py` — while 循环读取用户输入，调用 bot，流式打印回复
5. `storage.py` — 对话结束时将 messages 序列化为 JSON 存储到 `~/.patpat/history/`

## 6. 风险评估

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| LLM 回复不当，给出有害建议 | 高 | 严格 system prompt 约束 + 危机检测兜底 |
| 用户将 Bot 当作专业心理咨询替代品 | 中 | 每次启动时显示免责声明 |
| API 调用成本 | 中 | 默认使用 gpt-3.5-turbo，可配置切换 |
| API Key 泄露 | 中 | .env + .gitignore，不硬编码 |
| 对话上下文过长导致 token 超限 | 低 | 实现滑动窗口，保留最近 N 轮对话 |
